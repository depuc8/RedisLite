# RedisLite Benchmarks

> ⚠️ **AI-generated** — This benchmark suite was generated by an AI assistant. It is a standalone performance tool and is **not** part of the main RedisLite project build or CI pipeline.

Measures throughput (ops/sec) and latency (avg / p50 / p99) for all major RedisLite commands, plus pipeline speedup analysis.

---

## Prerequisites

- A running `redislite` server on `127.0.0.1:1234`
- `g++` with C++17 support

---

## Build & Run

```bash
# From this directory (benchmarks/)
make          # compiles → ./benchmark
make run      # compiles + runs (reminder to start server first)
make clean    # removes the binary
```

Or manually:

```bash
# Start the server from the project root first
../bin/redislite &

# Build and run the benchmark
make run
```

---

## What is measured

| Suite | Operations | Metric |
|---|---|---|
| **Single-shot** | SET, GET, DEL, KEYS | ops/sec · avg / p50 / p99 latency |
| **ZSet** | ZADD, ZQUERY | ops/sec · avg / p50 / p99 latency |
| **Pipeline** | SET at depth 1 / 10 / 100 | ops/sec · speedup vs depth=1 |

Default sample sizes: **N=50,000** for string ops, **N=20,000** for ZSet ops.

## Notes

- The benchmark connects a **single TCP connection** with no concurrent clients; results reflect single-client throughput.
- Each suite includes a **warmup phase** (200 ops) before timing begins.
- All response bytes are read and discarded (not parsed) to minimize client-side overhead.
